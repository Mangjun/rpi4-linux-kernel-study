# 📖 [커널 이론] ftrace 콘텍스트 플래그와 인터럽트 처리 아키텍처

## 1. ftrace 4가지 콘텍스트 기호 해부
ftrace 로그에 찍히는 4개의 알파벳(`d..1` 등)은 함수가 호출된 순간의 <b>커널 심장 상태(바이탈 사인)</b>를 의미한다.

* **1. irqs-off (인터럽트 잠금):**
  * `.` (정상): 하드웨어 인터럽트 수신 가능 상태.
  * `d` (disabled): CPU의 귀를 막아 외부 하드웨어 신호를 전면 차단한 상태. (지속될 경우 시스템 지연/Drop 발생)
* **2. need-resched (스케줄링 예약):**
  * `.` (정상): 현재 프로세스 계속 실행.
  * `N` (Need): 스케줄러가 현재 프로세스를 내리고 다른 타스크로 교체해야 한다고 깃발을 든 상태.
* **3. hardirq / softirq (인터럽트 콘텍스트):**
  * `.` (정상): 일반 프로세스 또는 커널 스레드 실행 중.
  * `H` (Hard IRQ): 하드웨어 인터럽트 핸들러가 CPU를 선점한 상태.
  * `S` (Soft IRQ): 지연된 인터럽트 후반부 처리(Bottom Half)가 실행 중인 상태.
* **4. preempt-depth (선점 카운터):**
  * `.` (0): 언제든 다른 프로세스에게 CPU를 양보(선점)할 수 있는 안전한 상태.
  * `1~N`: 스핀락(Spinlock) 등을 잡아 임계 영역(Critical Section)에 진입하여, 다른 프로세스가 절대 CPU를 뺏어갈 수 없는 선점 금지 상태.

## 2. 리눅스 커널의 '반갈죽' 인터럽트 처리 전략 (Top / Bottom Half)
고성능 네트워크/영상 장비에서 대량의 인터럽트가 쏟아질 때 시스템이 멈추지 않도록 하는 핵심 설계 철학.

* **1단계: Hard IRQ (Top Half) -> `H` + `d` 상태**
  * 신호가 들어오면 즉시 CPU의 귀를 닫고(`d`), 데이터의 메모리 주소만 넘기는 등 0.0001초 만에 끝나는 초간단 작업만 수행.
  * 무거운 작업은 Soft IRQ로 예약(Pending)하고 즉시 귀를 다시 연다(`.`).
* **2단계: Soft IRQ (Bottom Half) -> `S` + `.` 상태**
  * CPU 귀가 열려있는 상태(`.`)에서 패킷 파싱, 영상 메타데이터 조립 등 무거운 연산을 수행.
  * 이 연산 도중에도 귀가 열려 있으므로, 새로운 하드웨어 인터럽트가 들어오면 언제든 다시 접수할 수 있어 데이터 유실(Drop)을 완벽하게 방지한다.
